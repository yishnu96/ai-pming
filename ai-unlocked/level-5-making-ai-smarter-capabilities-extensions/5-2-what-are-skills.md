# What Are Skills?

You told the AI exactly how to format your weekly report last Monday. Then again on Tuesday. And again on Wednesday.

By Thursday, you were pasting the same thirty-line instruction block into the chat like a broken record. If you've done this, you're not alone. This is the single most common frustration people have with AI.

There's a better way. It's called Skills.

By the end of this article, you'll understand what Skills are, how they save your time, and where to find them across the AI tools you already use.

## The Copy-Paste Problem

Here's a scenario you've probably lived.

You want the AI to summarize your meeting notes. You've figured out exactly how to ask: include action items, flag risks, keep it under 200 words, use bullet points, skip the small talk.

It works great. The result is perfect.

Then next week, you need another meeting summary. You dig through your chat history, find that old prompt, copy it, paste it, modify the date, and send it off.

This repeats every single week. Sometimes every day.

::: The Pattern]
Every time you find yourself giving the AI the same instructions for the same kind of task, you've just identified something that should be a Skill. Skills solve the "I already told you this" problem permanently.
:::

Think of it like this. You wouldn't explain to a new employee how to run your weekly standup meeting and expect them to remember it without writing a procedure document. Then you'd hand that document to the next person, and the next. You've got a playbook.

That's all a Skill is: a playbook for your AI.

## What Are Skills

A Skill is a saved set of instructions you give your AI once, and it remembers them forever. Instead of typing the same prompt every time, you store it as a Skill and activate it with a word, a click, or a command.

::: Skills in One Sentence]
A Skill is a reusable instruction package — your best prompt, saved and ready to reuse anytime, across multiple conversations.
:::

Here's what that looks like in everyday work.

Say you're a product manager who writes user story summaries for your team. You've figured out the perfect formula: include the user persona, the pain point, the proposed solution, and the priority level. Written as a prompt, it takes thirty seconds to compose. You do this five times a week.

Over a year, that's roughly twenty minutes of your life spent typing the same thing over and over. A Skill cuts that to zero. You pick your "User Story Summary" Skill and type the raw notes. The AI already knows the format.

Skills work in two ways.

- **Automatic activation**: You set up a Skill, and the AI loads it on its own when it sees a matching task. You just describe your need — the instructions come included.
- **Manual activation**: You explicitly pick which Skill to use for a specific conversation. Good for tasks you do less often.

Either way, you write the instructions once. After that, every conversation inherits them.

## Skills vs. One-Time Prompts

It's worth clarifying the difference, because Skills build on everything you learned about prompting.

| One-Time Prompt | Skill |
| — | — |
| Type it fresh each time | Write it once, reuse forever |
| Lives in your chat history | Saved and organized |
| Forgets when the chat ends | Persists across conversations |
| You control every detail each session | The AI applies your rules automatically |

Think of a prompt as speaking. Think of a Skill as teaching the AI a habit. Once the habit is formed, you don't have to keep repeating yourself.

## Where Skills Live

Skills aren't a single product. They're a capability that shows up across different AI tools — each with its own flavor, same core idea.

### Claude Code

Claude Code lets you create custom Skills that live in your project or on your computer. When Claude starts working, it scans your available Skills and automatically picks the right one for the task.

For example, a "Brand Review" Skill could instruct Claude to check every piece of copy against your company's voice and tone guidelines. It loads automatically whenever you ask Claude to review content.

Claude also ships with built-in Skills for common tasks. You don't need to build everything from scratch.

::: Where Skills Are Stored in Claude Code]
Claude Code checks four locations for Skills, in priority order: enterprise level (company standards), personal level (your preferences), project level (team-specific rules), and plugin level (community additions). Higher-level Skills override lower ones.
:::

### Claude Projects

Claude Projects on the web (claude.ai) gives you a similar benefit without needing to set up files or folders.

When you create a Project, you add custom instructions that persist across every conversation inside that Project. You upload reference documents, set rules, and Claude remembers them in every new chat.

The difference from Claude Code Skills: Projects provide **context** (who you are, what you're working on, what documents to reference), while Skills provide **capabilities** (step-by-step procedures the AI follows). Think of Projects as the workspace and Skills as the toolkit inside it.

::: Available Plans]
Claude Projects with custom Skills are available on Pro, Max, Team, and Enterprise plans. If you're on the free plan, you can still use Projects — you just won't have the custom Skills upload feature.
:::

### Custom GPTs (OpenAI)

OpenAI's equivalent is Custom GPTs. You create a tailored version of ChatGPT with specific instructions, knowledge files, and behaviors. It gets its own name and description.

For example, a marketing team might create a "Brand Copy Checker" GPT that knows their style guide, preferred tone, and formatting rules. Anyone on the team can use it without writing any instructions themselves.

Custom GPTs can also be shared across teams or made public. If you build something useful, others can benefit from it too.

### The Big Picture

All three platforms — Claude Code, Claude Projects, and Custom GPTs — solve the same fundamental problem: stop repeating yourself. The implementation differs, but the value proposition is identical.

| Platform | Best For | Skill Type |
| — | — | — |
| Claude Code | Deep, project-based work | Automatic skill loading from files |
| Claude Projects | Everyday web chat with persistent context | Custom instructions + reference files |
| Custom GPTs | Shared team tools anyone can access | Standalone customized AI assistants |

## Try This Now

Open Claude, ChatGPT, or any AI tool you use regularly. This takes five minutes.

**Step 1:** Think of one task you've given the AI similar instructions for more than twice. Maybe it's summarizing emails, formatting meeting notes, or drafting status updates.

**Step 2:** Write down the complete instructions you'd ideally give — all the context, constraints, and format rules you usually type from memory.

**Step 3:** In Claude Projects, create a new Project for that specific work area. Add your instructions as the custom instructions for that Project. Give it a clear name you'll remember.

**Step 4:** Start a new conversation inside that Project. Give it a simple task related to that area. Notice how the AI already knows what you want — no copy-pasting needed.

That's your first Skill in action. You'll never type those instructions again.
